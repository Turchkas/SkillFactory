# Semantic Hallucination Detector

Проект предназначен для автоматического обнаружения семантических галлюцинаций в русскоязычных текстах на основе комбинированного подхода, включающего извлечение утверждений, поиск доказательств и логический вывод (Natural Language Inference).

---

## 📌 Архитектура системы

Система реализует следующий пайплайн:

1. **Извлечение утверждений (Claim Extraction)**
   Текст разбивается на отдельные предложения с использованием библиотеки `natasha`.

2. **Поиск доказательств (Retrieval)**
   Для каждого утверждения выполняется поиск релевантных текстовых фрагментов:

   * используется модель эмбеддингов
   * применяется FAISS для быстрого поиска
   * возвращаются top-k наиболее релевантных предложений

3. **Логическая проверка (NLI)**
   Для каждой пары *(утверждение — доказательство)* определяется отношение:

   * entailment (подтверждение)
   * contradiction (противоречие)
   * neutral (неопределённость)

4. **Принятие решения**
   Утверждение считается **галлюцинацией**, если ни одно доказательство не даёт entailment.

## 🧪 Эксперимент 1 (глава 4)

Запуск:

```bash
python evaluate_metrics.py
```

### Что происходит:

* загружается датасет `nli_fever-ru_10k`
* формируется корпус из `premise`
* для каждого утверждения выполняется:

  * retrieval
  * NLI
  * классификация

### Рассчитываются метрики:

* Precision
* Recall
* F1
* Accuracy

### Результаты:

📄 `cache/metrics_results.json`
→ используется для таблицы 4.1

---

## 🔍 Анализ ошибок

Дополнительно сохраняется:

📄 `cache/detailed_results.json`

Содержит:

* утверждение
* предсказания
* найденные доказательства
* автоматическую классификацию причины ошибки

### Типы ошибок:

* retrieval_error
* nli_error
* paraphrase_or_lexical_gap

⚠️ Классификация является эвристической и используется для последующего ручного анализа.

## 🧪 Эксперимент 2 (LLM)

Файл:

```bash
python llm_experiment.py
```

### Особенности:

* небольшой набор текстов (имитация LLM)
* ручная разметка
* бинарная классификация:

  * 0 — корректно
  * 1 — галлюцинация

---

## 🧠 Используемые модели

### Embeddings:

* `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`

### NLI:

* `cointegrated/rubert-base-cased-nli-threeway`

---

## 📊 Датасет

* `MilyaShams/nli_fever-ru_10k`

Используется:

* premise → корпус
* hypothesis → утверждения

---

## ⚠️ Ограничения

* зависимость от качества retrieval
* ограниченность корпуса (Wikipedia-based)
* чувствительность к переформулировкам
* бинаризация neutral → галлюцинация
